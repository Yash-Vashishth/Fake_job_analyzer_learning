Scoring Methodology

Formula Reference

Final Score Formula
FinalScore = clamp(WeightedBase + BoosterTotal, 0, 100)
Step 1 — Weighted Base
For each active parameter p:
  fraud_contribution(p) = (1 − score(p)/10) × weight(p)

WeightedBase = Σ fraud_contribution(p) / Σ weight(p) × 100
Score 10/10 = perfectly legitimate (zero fraud contribution). Score 0/10 = maximum fraud signal.
Step 2 — Hard Gate Boosters
Hard gates: causal_inversion, tool_anachronism, domain_legitimacy
  Threshold: score ≤ 2

  If triggered → penalty = weight × 1.2 (added directly to final)
  These BYPASS the weighted average entirely when they fire.
Rationale: a caught typosquat domain or timestamp contradiction is a near-certain fraud signal regardless of what other parameters show.
Step 3 — Mid-Tier Booster
Boosters: urgency_language
  Threshold: score ≤ 3

  If triggered → penalty = weight × 0.8 (additive)
               + remains in weighted formula
Urgency/payment-request language is highly predictive but not binary — it also stays in the weighted average to avoid over-penalizing mild urgency.

About This Framework

Overview

This tool implements a multi-modal forensic framework for detecting fake job offers, with a specific focus on India-specific fraud patterns (MCA21/CIN cross-reference, payment-request scams, typosquatting of Indian corporates).

Analysis Pipeline

For each input, the system runs up to 5 forensic modules in parallel. Algorithmic scores from signal processing (DCT, Laplacian, Levenshtein) are then enriched by an LLM pass (Claude Sonnet) for semantic parameters. The final score is computed via the 3-layer weighted formula with hard-gate boosters.

Limitations

Pixel-level parameters (glyph sharpness, DCT, ink entropy) are marked N/A when only text input is provided. Cross-platform consistency is heuristic — full verification requires live portal API access. Always supplement HIGH RISK verdicts with MCA21 CIN lookup and cybercrime.gov.in reporting.
